grok-cli

AI coding agent powered by xAI Grok — with persistent memory, security hardening, and custom sub-agents

Install

curl -fsSL https://raw.githubusercontent.com/alphaonedev/grok-cli/main/install.sh | bash

What's Different in This Fork

Persistent Memory

ai-memory MCP integration with session-scoped connections, auto-recall on startup, and compaction summary storage. Your agent remembers across sessions.

Security Hardening

MCP tool name collision prevention (39 reserved names), description sanitization against prompt injection, proper lifecycle cleanup.

10 Sub-Agents

architect, code-review, debug, implement, refactor, test-writer, docs, quick-fix, data-ops, security-audit — across 3 model tiers.

All-Mode MCP

MCP tools work in plan, ask, and agent modes. Memory recall available everywhere, not just when executing tasks.

Pinned Dependencies

Exact version pins for @ai-sdk/xai, ai, and zod prevent bun cache corruption that caused the 410 Live Search bug.

Full Grok Model Lineup

grok-4.20-reasoning ($2/M), grok-4-1-fast-reasoning ($0.20/M), and 13 more models. All with 2M token context.

ai-memory Integration (optional)

Add persistent cross-session memory. Merge into the config above:

{
  "mcp": {
    "servers": [{
      "id": "ai-memory",
      "label": "AI Memory",
      "enabled": true,
      "transport": "stdio",
      "command": "ai-memory",
      "args": ["mcp", "--tier", "semantic"]
    }]
  }
}
